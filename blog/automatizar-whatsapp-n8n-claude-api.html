Cómo automatizar WhatsApp
para empresas con n8n y Claude API

La mayoría de los tutoriales de WhatsApp + IA muestran demos que nunca llegan a producción. Este artículo es diferente: es la arquitectura real que uso en Luca y Valentina, dos asistentes personales con usuarios de pago activos.

Por qué WhatsApp y no otra app

Cuando una empresa me pregunta por dónde empezar con IA conversacional, casi siempre la respuesta es WhatsApp. No por moda — por datos. España tiene una tasa de penetración de WhatsApp superior al 90% en adultos. Tu cliente ya lo usa, ya lo tiene instalado, ya sabe cómo funciona. No tienes que convencerle de nada.

Un agente en WhatsApp no necesita app propia, no necesita onboarding complicado y funciona en cualquier dispositivo. El coste de adquisición es casi cero comparado con una app nativa.

Los tres componentes de la arquitectura

Un agente IA en WhatsApp tiene tres piezas que deben encajar perfectamente:

  • WhatsApp Business API (Meta) — el canal. Gestiona la recepción y envío de mensajes. Necesitas una cuenta de Meta for Developers y un número de teléfono dedicado.
  • n8n — el cerebro orquestador. Recibe el webhook de Meta, procesa la lógica, llama a la IA y devuelve la respuesta. Es tu capa de automatización y memoria.
  • Claude API (Anthropic) — la inteligencia. Procesa el mensaje, entiende el contexto y genera una respuesta natural. Claude es especialmente bueno siguiendo instrucciones complejas sin alucinaciones.

El flujo paso a paso

1. Configura el webhook de Meta

Meta envía cada mensaje entrante a una URL (tu webhook). En n8n creas un nodo Webhook con método POST y esa URL la registras en el panel de Meta for Developers, en la configuración de tu app de WhatsApp Business.

URL webhook: https://tu-n8n.com/webhook/whatsapp-entrada Método: POST Token verificación: [genera uno aleatorio y guárdalo]

2. Parsea el mensaje entrante

Meta envía un JSON con bastante ruido. Lo primero que hace tu workflow de n8n es extraer lo que importa: el número de teléfono del remitente, el texto del mensaje y el timestamp.

// Nodo Function en n8n const body = $input.first().json.body; const entry = body.entry[0].changes[0].value; const message = entry.messages[0]; return { phone: message.from, text: message.text.body, timestamp: message.timestamp };

3. Recupera el historial de conversación

Aquí está la diferencia entre un chatbot básico y un agente real. El agente recuerda. Guarda cada mensaje en PostgreSQL con el número de teléfono como identificador y recupera los últimos N mensajes antes de llamar a la IA. Así el agente mantiene el contexto de la conversación.

4. Llama a Claude API con el contexto

El nodo HTTP de n8n hace la llamada a la API de Anthropic con el historial como contexto y el mensaje nuevo como último turno del usuario.

POST https://api.anthropic.com/v1/messages Authorization: x-api-key: TU_API_KEY anthropic-version: 2023-06-01 { "model": "claude-sonnet-4-20250514", "max_tokens": 1024, "system": "Eres [nombre del asistente]. Tu objetivo es...", "messages": [ ...historial_previo, {"role": "user", "content": "mensaje_del_usuario"} ] }

5. Envía la respuesta por WhatsApp

Con la respuesta de Claude, haces otra llamada HTTP a la API de Meta para enviar el mensaje al número de origen.

POST https://graph.facebook.com/v18.0/{PHONE_ID}/messages Authorization: Bearer {TOKEN} { "messaging_product": "whatsapp", "to": "{phone_number}", "type": "text", "text": {"body": "{respuesta_de_claude}"} }

Lo que aprendo cada día en producción real

Llevar Luca y Valentina con usuarios de pago me ha enseñado cosas que no aparecen en ningún tutorial:

  • Gestiona los duplicados de webhook — Meta a veces envía el mismo mensaje dos veces. Guarda el message_id y descarta duplicados antes de procesarlos.
  • Timeouts de 20 segundos — Meta espera como máximo 20 segundos tu respuesta al webhook o lo marca como fallido. Si Claude tarda más (raro pero pasa), responde primero con un mensaje de espera y envía la respuesta real después.
  • Limita el historial — Enviar toda la conversación a Claude se vuelve costoso y lento. Los últimos 10-15 mensajes son suficientes para el 99% de los casos.
  • Separa los workflows por número — Si tienes más de un asistente (como yo con Luca y Valentina), usa un único webhook de entrada con routing inteligente por número de teléfono.

Cuánto cuesta en producción

La pregunta que siempre hacen. Con un volumen medio-bajo (1.000 mensajes/mes por usuario activo):

  • WhatsApp Business API — gratuito hasta 1.000 conversaciones/mes, luego ~0,05€ por conversación de servicio.
  • Claude API — con claude-sonnet ~0,003€ por mensaje estándar. 10.000 mensajes al mes son unos 30€.
  • n8n self-hosted — gratuito si lo montas en tu propio VPS (5-10€/mes un VPS básico).
  • PostgreSQL — incluido en el VPS.

El coste real de un agente en producción para 100 usuarios activos es menos de 80€ al mes en infraestructura.

Conclusión: La arquitectura WhatsApp + n8n + Claude es la más rápida de llevar a producción, la más barata de mantener y la que mejor funciona en el mercado español. No necesitas ser desarrollador experto — necesitas entender el flujo y tener el tiempo de montarlo bien desde el principio.
